Hackathon Project Plan · No Headset Required

CitizenLens

Mobile AR + WebXR platform that lets citizens spatially experience urban planning decisions before construction begins — no VR headset needed.

Domain Urban Planning & Infrastructure
Platform Mobile AR + Web Browser
Duration 48 Hours
XR Method WebXR / Three.js / 8th Wall
01 — The Problem
Pain Point #1
Zero Spatial Empathy
City councils approve buildings from 2D blueprints. Residents can't visualize scale, shadow, or street-level impact until construction is already done.
Pain Point #2
Late-Stage Protests
Community backlash only happens after approvals. Objections that could have shaped design are raised too late — costing time, money, and trust.
The Gap We Fill
No tool lets ordinary citizens walk around a proposed building in the real world, on their own phone, before a single brick is laid.
CitizenLens bridges spatial understanding and civic participation — using mobile AR, anyone can point their phone at an empty plot and see the future, life-size.
02 — No-Headset Adaptation
⚡
Plan Adapted: VR → Mobile AR + WebXR Since your team has no VR headset, we're switching to a phone-first AR approach using WebXR and Three.js — which actually has a BIGGER demo advantage: judges can try it on their own phones instantly, no hardware needed.

Meta Quest / PC VR
Required hardware, limited to team's device, difficult for judges to try, complex setup time during demo.
✅ New Plan (Mobile AR)

03 — What You Build (MVP)
★ Core Product
CitizenLens AR Visualizer
"Point. See. Decide."
A mobile web app where users tap a real-world location on a map, then point their phone camera at that spot to see a 3D building model placed there at real-world scale — with shadow simulation, height indicators, and a community vote button.
01
Map-Based Plot Selector
User opens the web app, sees a Mapbox map with pre-loaded "proposed project" pins. Tap a pin to select the building to visualize.
02
Mobile AR Placement
After selecting, the app switches to camera mode. Using WebXR + device sensors, the 3D model is placed on the ground at the correct GPS location and scale.
03
Shadow Simulation Slider
A time-of-day slider shows how the building's shadow falls across the neighborhood at 8AM, 12PM, and 5PM using Three.js directional light simulation.
04
Citizen Vote + Comment
While in AR view, a floating UI lets citizens tap "Support", "Object", or "Suggest Change" — pinned to that exact AR location with a counter display.
05
Admin Dashboard (Web)
A simple web dashboard shows planners a heatmap of citizen reactions overlaid on the city map, with a breakdown of votes per project.
06
QR Code Share
Each project generates a shareable QR code so citizens can be directed to the exact AR experience — perfect for the demo moment with judges.
04 — Tech Stack (No Headset)
XR / 3D Layer
Three.js + WebXR
Render 3D building models in browser. WebXR API handles phone camera AR. Works on Android Chrome natively. iOS via WebXR Viewer app.
Alternative XR
8th Wall (Free Trial)
If WebXR proves tricky, 8th Wall provides instant cross-platform AR with surface detection. Free 14-day trial — perfect for a hackathon.
3D Models
glTF + Sketchfab
Download free building models from Sketchfab or Poly Pizza in glTF format. Three.js loads these natively. No 3D modeling needed.
Maps
Mapbox GL JS
Free tier. 3D terrain, building footprint data, and GPS coordinate picking built-in. Drop a pin → get real-world coordinates for AR anchor.
Backend / DB
Firebase / Supabase
Store citizen votes and comments in real-time. Free tier sufficient. No backend code needed — direct SDK from frontend.
AI Layer
Claude API
Summarize citizen feedback clusters into actionable planning insights. "Citizens near Block C mostly object due to shadow impact on the park."

06 — 48-Hour Build Timeline
Hour 0 – 3 · Setup Sprint
Foundations & Repo
  • Initialize Vite + Three.js project, deploy to Vercel
  • Set up Firebase project, configure Mapbox API key
  • Download 2–3 glTF building models from Sketchfab
  • Lock on tech: WebXR native vs 8th Wall (test both, pick one)
Hour 3 – 12 · Core XR Build
AR Scene Working
  • Get WebXR camera session running in mobile browser
  • Load glTF model, place it on detected ground plane
  • Connect Mapbox pin → pass GPS coordinates to AR scene
  • Basic hit-testing: model sticks to real floor surface
Hour 12 – 24 · Feature Layer
Shadow Sim + Voting UI
  • Implement directional light with time-of-day slider
  • Build floating AR overlay panel: vote buttons
  • Wire vote button → Firebase write → live counter update
  • Admin dashboard page: read vote data, show summary
Hour 24 – 36 · AI + Polish
Intelligence & UX
  • Integrate Claude API: summarize vote comments into insights
  • Polish mobile UI: touch controls for model scale/rotate
  • Add QR code generator for each project URL
  • Test on multiple phones — Android + iOS
Hour 36 – 44 · Demo Prep
Rehearse & Harden
  • Fix critical bugs only — freeze feature scope
  • Load a real local landmark as the "proposed site"
  • Rehearse 3-min live demo: map → AR → vote → AI insight
  • Prepare offline fallback (pre-recorded video backup)
Hour 44 – 48 · Pitch
Present & Win
  • 5-slide deck: Problem → Solution → Demo → Impact → Ask
  • Hand judges the QR code before you start talking
  • Show AI feedback summary live on the dashboard
  • Close with: "This city. This plot. This building. Right now."
